# SEO Optimization Plan for Static Hosting (GitHub Pages)

Этот документ описывает стратегию SEO-оптимизации для проекта FinCalc, учитывая его размещение на статическом хостинге.

## 1. Техническое SEO (Technical SEO)

### 1.1. Структура URL и редиректы
- **Текущее состояние:** URL чистые (`.html`), структура папок логичная (`/articles/`, `/calculators/`).
- **Задача:** GitHub Pages не поддерживает серверные редиректы `.htaccess`.
- **Решение:** Использовать страницу `404.html` для обработки ошибок. (Выполнено)

### 1.2. Sitemap.xml и Robots.txt
- **Sitemap:** Должен содержать все важные страницы с приоритетами и датами обновления.
  - *Важно:* При добавлении новых статей нужно вручную обновлять `sitemap.xml` или использовать скрипт генерации.
- **Robots.txt:** Правила уже настроены корректно. Блокирует логи и системные файлы.

### 1.3. Canonical Links
- Критически важно для многоязычного сайта.
- Каждая страница должна иметь `<link rel="canonical" href="..." />`, указывающий на саму себя (или основную версию).
- Теги `hreflang` должны связывать все языковые версии одной страницы.

## 2. On-Page SEO

### 2.1. Мета-теги (Meta Tags)
Для каждой страницы необходимо заполнить:
- `<title>`: Уникальный, содержит ключевые слова (50-60 символов).
- `<meta name="description">`: Привлекательное описание для сниппета в выдаче (150-160 символов).
- **Open Graph (OG):** Для красивого отображения в соцсетях (Facebook, LinkedIn, мессенджеры).
  - `og:title`
  - `og:description`
  - `og:image` (ссылка на превью)
  - `og:url`
  - `og:type`
- **Twitter Cards:**
  - `twitter:card`
  - `twitter:title`
  - `twitter:description`
  - `twitter:image`

### 2.2. Структурированные данные (Schema.org)
Используем JSON-LD для помощи поисковикам в понимании контента.
- **Главная:** `WebSite`, `Organization`, `SoftwareApplication`.
- **Статьи:** `Article` или `BlogPosting`.
- **Калькуляторы:** `SoftwareApplication` или `WebApplication`.
- **Хлебные крошки:** `BreadcrumbList`.

### 2.3. Контент
- Использовать семантические теги: `<header>`, `<main>`, `<article>`, `<footer>`, `<aside>`, `<nav>`.
- Заголовки `<h1>` (один на страницу), `<h2>`, `<h3>` должны выстраивать четкую иерархию.
- Изображения должны иметь атрибут `alt` с описанием.
- Для оптимизации загрузки изображениям добавляем `loading="lazy"`.

## 3. Производительность (Performance)
На статике это особенно важно.
- Минификация CSS/JS (рекомендуется перед деплоем).
- Использование современных форматов картинок (WebP).
- Отложенная загрузка сторонних скриптов (Analytics и т.д.).

2. OG и Twitter разметка.
3. Schema.org JSON-LD (`Article`, `BreadcrumbList`).
